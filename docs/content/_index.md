The `docs/` directory is now the shared source for three things:

1. Human-readable project documentation.
2. A Hugo site that can publish to GitHub Pages.
3. LLM handoff material for future contributors and agents.

## Start here

- [Project docs](./docs/) for architecture, backend, frontend, UI, and reference docs.
- [LLM handoff](./llm/) for concise project context and continuation guidance.
- [Published frontends](#frontend-builds) for static builds copied from workspace apps during `npm run docs:prepare`.

## Local docs workflow

```bash
npm run docs:prepare
npm run docs:dev
npm run docs:build
```

`docs:prepare` rebuilds frontend workspaces and copies any `apps/*/dist/` output into the docs site under `/apps/<workspace>/`.
It also regenerates `docs/data/generated/frontends.json`, which Hugo uses to list published frontends on the home page.

## Generated docs assets

- `docs/static/apps/` is generated by `npm run docs:prepare`.
- `docs/data/generated/frontends.json` is generated by the same step and is part of the docs build logic.
- These generated files should not be edited by hand.

## Historical notes

Some pages under `docs/content/docs/` are intentionally archival or context-oriented. Current examples called out in the docs themselves include:

- `discussion-summary.md` — earlier architecture discussion background
- `MIGRATION.md` — only the remaining migration follow-up and long-term backend direction

When in doubt, prefer the operational docs such as `architecture`, `frontend-react`, `frontend-svelte`, `ui-conventions`, and the LLM context pages over background material.

## Current UI target

The next major UI migration target is the React `Aspectarium`.

- Vision source: `inspiration-source/aspektarium/Kefer - Aspektárium/`
- Working implementation file: `apps/web-react/src/app/components/aspectarium.tsx`
- Normative UI rules live in [UI conventions](./docs/ui-conventions/)

Current target behavior:

- rows come from the selected objects-to-compute list, like the radix position report
- the diagonal repeats the same objects; transit/event-range mode may expand this into a square
- clicking a computed aspect cell opens the detail view on the right
- desktop layout should default to an 80/20 matrix-to-detail split
- shared shadcn-style primitives should be reused before introducing new bespoke UI
